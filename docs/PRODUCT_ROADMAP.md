# Visual Vocab — 제품 고도화 로드맵

> **포지셔닝 한 문장:** "상상 속 회화를 연습하는 앱이 아니라, **지금 눈앞의 상황에서 말문을 트게 하는** 앱."
>
> 회화 앱은 넘쳐나지만, 대부분 가상의 시나리오를 전제로 한다. Visual Vocab은 반대로 간다 —
> 사용자가 실제로 서 있는 그 장소를 카메라로 비추면, 그 자리에서 필요한 단어와 한 마디가 나온다.
> "단어가 생각나지 않아 아예 물어보지 못하는" 순간을 없애는 것이 이 제품의 존재 이유다.

---

## 1. 타겟 페르소나

### 페르소나 A — 여행객 (1차 타겟, 수익화 대상)
- **상황:** 일본 편의점, 대만 야시장, 스페인 약국. 사고 싶고 묻고 싶은데 그 물건의 이름을 모른다.
- **핵심 니즈:** 지금 당장. 학습이 아니라 **구조(rescue)**. 3초 안에 단어가 나와야 한다.
- **필요 기능:** 빠른 스캔, 큰 글씨 발음(한글 표기), TTS "대신 말해주기", 오프라인 대비, 로밍 데이터 절약.

### 페르소나 B — 어린이 학습자 (2차 타겟, 리텐션 대상)
- **상황:** 집·유치원·놀이터에서 부모와 함께 사물을 찍으며 단어 놀이.
- **핵심 니즈:** 재미와 반복. 학습이 놀이처럼 느껴져야 한다.
- **필요 기능:** 키즈 모드(큰 라벨·쉬운 단어 우선·밝은 애니메이션), 스티커/도감 수집 메타포, 부모용 진도 확인.

### 공통 인사이트
두 페르소나 모두 "교재 속 단어"가 아니라 "**내 눈앞의 단어**"를 원한다.
차별화 메시지는 하나로 통일한다: **"세상이 곧 단어장이다."**

## 2. 현재 상태 진단 (2026-07 기준)

**되는 것 (핵심 루프 완성)**
- 사진/카메라 → 사물 인식 → 언어별 번역 라벨 + 바운딩 박스 오버레이
- 로마자 + 한글 외래어 발음 표기 (일본어·중국어)
- 라이브 모드(실시간 연속 분석), TTS 발음 재생, 단어 저장
- 인식된 사물 기반 상황 추론 → 실전 회화 문장 5개 추천

**비어 있는 것 (고도화 대상)**
| 갭 | 문제 | 우선순위 |
|---|---|---|
| UI 문구가 전부 영어 | 타겟이 한국어 화자인데 "No saved words yet" | P0 |
| 학습 루프 부재 | 저장 탭이 단순 리스트 — 복습·퀴즈·망각곡선 없음 | P0 |
| 온보딩 부재 | 첫 실행 시 앱의 가치를 설명하는 화면 없음 | P1 |
| 키즈 모드 없음 | 페르소나 B를 받을 그릇이 없음 | P1 |
| 오프라인 미대응 | 여행지에서 네트워크 없으면 저장 단어 열람만 가능해야 하는데 그마저 검증 안 됨 | P1 |
| DB 미사용 | `lib/db`(Drizzle)가 있지만 실제 저장은 로컬 — 계정/동기화 없음 | P2 |
| 비용 통제 없음 | 라이브 모드가 API를 연속 호출 — 사용자당 비용 상한 필요 | P0 (배포 전 필수) |

## 3. UX 고도화 로드맵

### Phase 1 — "출시 가능한 앱" (2~3주)
1. **전면 한국어화** — 모든 UI 문구, 알림, 에러 메시지. i18n 구조(ko 기본, en 추가)로.
2. **온보딩 3장** — ① "찍으면 단어가 보여요" ② "그 상황의 한 마디까지" ③ 카메라 권한 요청. 첫 스캔까지 30초 안에 도달하게.
3. **구조 모드(Rescue) UX** — 홈에서 한 번의 탭으로 스캔 → 결과 화면에서 단어를 탭하면 **크게 보기 + TTS 재생** (점원에게 화면을 보여주거나 소리를 들려주는 용도). 여행객 페르소나의 킬러 기능.
4. **복습 루프 v1** — 저장 단어에 간단한 SRS(오늘 복습할 단어 N개 → 사진 보고 단어 맞히기 퀴즈). 찍은 사진의 크롭을 퀴즈 카드로 재사용하면 "내가 찍은 단어장"이라는 정체성이 강해진다.
5. **비용 가드레일** — 라이브 모드 호출 간격 제한, 일일 무료 스캔 횟수 제한(프리미엄 해제), 이미지 리사이즈 최적화.

### Phase 2 — "페르소나 분화" (4~6주)
1. **키즈 모드** — 토글 하나로: 쉬운 단어 우선 프롬프트, 큰 라벨·큰 버튼, 수집 도감(찍은 사물이 스티커로 쌓임), 효과음. 부모 설정에서 학습 리포트.
2. **여행 모드** — 국가 선택 시 해당 언어 자동 설정 + 상황 회화를 여행 시나리오(주문·계산·길묻기·응급)로 편향. 자주 쓰는 생존 표현 오프라인 팩 다운로드.
3. **오프라인 대응** — 저장 단어·오디오는 완전 오프라인 열람. 스캔 불가 시 안내 UX.
4. **계정 & 동기화** — 이미 있는 `lib/db`(Drizzle + Postgres)를 활성화. 소셜 로그인, 기기 간 단어장 동기화.

### Phase 3 — "차별화 굳히기" (그 이후)
- 발음 따라 말하기(STT 채점), 가족 공유 단어장, 주간 리포트("이번 주 당신의 세상에서 배운 단어 23개"), 위젯(오늘의 복습 단어).

## 4. 배포 계획

### 앱 배포
- **Expo EAS Build** 로 iOS/Android 동시 빌드. `eas.json` 추가 → TestFlight/내부 테스트 트랙으로 베타 → 스토어 심사.
- 심사 대비 체크: 카메라/사진 권한 문구(이미 있음 ✓), 개인정보처리방침 URL(필요), 아동 대상 기능이 있으면 스토어의 아동 관련 정책(광고·데이터 수집 제한) 검토 필수.

### 서버 배포
- 현재 Express API는 Replit 호스팅 전제. 초기엔 Replit Deployments로 충분하고, 트래픽이 붙으면 Fly.io/Railway 등으로 이전 용이(esbuild 단일 번들이라 이식 쉬움).
- **필수 선행:** API 인증(앱 토큰) 추가 — 지금은 엔드포인트가 열려 있어 배포 시 비용 폭탄 위험.

### 프라이버시
- 사진이 OpenAI API로 전송됨 — 개인정보처리방침에 명시하고, 서버에 이미지를 저장하지 않는 현재 구조를 유지·명문화.

## 5. 마케팅 계획

### 메시지
- 메인 카피: **"말문이 막히는 순간, 카메라를 켜세요."**
- 서브: "상상 회화 말고, 지금 눈앞의 단어."

### 채널 (제품 특성상 데모가 곧 광고)
1. **숏폼 (핵심 채널)** — 이 앱은 15초 데모가 완벽하게 성립한다: 야시장에서 폰을 비추면 단어가 뜨는 화면 녹화 + "이거 뭐라고 물어봐요?" 훅. 릴스/쇼츠/틱톡에 상황별 시리즈(편의점편, 약국편, 공항편).
2. **여행 커뮤니티** — 여행 카페·오픈채팅에 "일본 여행 필수앱" 류의 실사용 후기 포맷.
3. **육아 커뮤니티** — 키즈 모드 출시 시점에 맘카페·인스타 육아 계정 협업("아이와 집안 물건 영어 놀이").
4. **ASO** — 키워드: 여행 회화, 사진 번역, 영어 단어장, 아이 영어. 스크린샷 첫 장은 무조건 라벨 오버레이 화면.

### 수익 모델 (제안)
- **프리미엄(구독):** 무료 = 일 N회 스캔 + 단어장. 유료 = 무제한 스캔, 라이브 모드, 오프라인 팩, 키즈 도감.
- API 원가가 스캔당 발생하므로 무료 티어 상한이 곧 비용 통제 장치.

## 6. 바로 다음 액션 (제안 순서)

1. Phase 1의 P0 세 가지: **한국어화 → 복습 루프 v1 → 비용 가드레일(API 인증 포함)**
2. `eas.json` 세팅 + TestFlight 베타 (지인 10~20명, 여행 예정자 우선)
3. 베타 피드백으로 구조 모드 UX 다듬기 → 스토어 출시
4. 출시와 동시에 숏폼 3편(편의점·카페·약국) 업로드 시작
